# Cluster-Based Optimal Data Selection for Continual Pre-training

基于聚类的持续预训练数据选择框架。通过 PMP (Perturbation-based Meta-Policy) 动态调整各聚类的采样权重，使训练数据分布自适应优化。

## 项目结构

```
.
├── train.py                  # 入口脚本
├── configs/
│   ├── default.yaml          # 全部超参数配置
│   ├── ds_zero3.json         # DeepSpeed ZeRO-3 配置 (纯 GPU)
│   └── ds_zero3_offload.json # DeepSpeed ZeRO-3 + CPU Offload 配置
├── data/
│   ├── json_dataset.py       # JsonFolderDataset — 通用 JSON/JSONL 数据加载
│   ├── eval_dataset.py       # FewShotEvalDataset — Few-shot / Zero-shot MCQ 评估
│   └── cluster_dataset.py    # ClusterDataset + ClusterWeightedSampler
├── clustering/
│   ├── kmeans_clusterer.py   # KMeans 聚类 (projection / embedding / ghost 特征)
│   └── random_clusterer.py   # 随机聚类 (baseline)
├── pmp/
│   ├── grad_utils.py         # Dev 梯度计算、JVP 聚类贡献度、Ghost Inner Product
│   ├── model_wrapper.py      # TransformerWrapper (参数向量化操作)
│   └── projection.py         # 随机投影 (Rademacher / Gaussian)

│   └── ring_buffer.py        # 环形缓冲区 (存储近期参数快照)
├── utils/

├── valid/                    # 验证集 (MMLU 格式)
├── dataset/                  # 训练数据

```

## 快速开始

### 安装依赖

```bash
pip install -r requirements.txt
```

### 单卡训练

```bash
python train.py --config configs/default.yaml
```

### 多卡训练

```bash
torchrun --nproc_per_node=8 train.py --config configs/default.yaml
```

### DeepSpeed ZeRO-3 训练 (默认)

```bash
deepspeed --num_gpus=8 train.py --config configs/default.yaml
```

默认配置已启用 DeepSpeed ZeRO-3，适合大模型训练 (如 Qwen3-8B)。

### DeepSpeed ZeRO-3 + CPU Offload

显存不足时，将优化器状态和参数 offload 到 CPU：

```bash
deepspeed --num_gpus=8 train.py --config configs/default.yaml \
    deepspeed.config_file=configs/ds_zero3_offload.json
```

### 关闭 DeepSpeed (使用 DDP)

```bash
torchrun --nproc_per_node=8 train.py --config configs/default.yaml \
    deepspeed.enabled=false
```

### CLI 覆盖参数

使用 OmegaConf dot-list 语法在命令行覆盖任意配置项：

```bash
python train.py --config configs/default.yaml \
    model.path=Qwen/Qwen2-7B \
    training.lr=1e-5 \
    clustering.method=random \
    data.eval_format=fewshot \
    data.n_shot=5
```

## 核心流程

```
训练数据 ──→ 特征提取 ──→ KMeans 聚类 ──→ 按权重采样 ──→ LM 训练
                                              ↑
                           PMP backward ──→ 更新聚类权重
                               ↑
                          验证集 dev gradient
```

每 `pmp.update_interval` 步执行一次 PMP 更新：
1. 在环形缓冲区中回溯最近 `window_size` 步
2. 计算验证集梯度 `nabla L_dev`
3. 通过 JVP 计算各聚类对验证 loss 的贡献度
4. 用 softmax 将贡献度转换为采样权重

## 评估

支持三种评估模式，通过 `data.eval_format` 配置切换：

### Few-shot MCQ 评估 (推荐)

```yaml
data:
  eval_format: "fewshot"
  n_shot: 5          # few-shot 示例数量
```

从验证集中选取 N 条同类示例拼接到 prompt 前面，模型预测 A/B/C/D 选项，输出 **accuracy**。

示例 prompt (n_shot=2):
```
The following are multiple choice questions (with answers).

Question: The cyclic subgroup of Z_24 generated by 18 has order
A. 4
B. 8
C. 12
D. 6
Answer: A. 4

Question: Find the order of the factor group Z_6/<3>.
A. 2
B. 3
C. 6
D. 12
Answer: B. 3

Question: Find the maximum possible order for some element of Z_4 x Z_6.
A. 4
B. 6
C. 12
D. 24
Answer:
```

### Zero-shot MCQ 评估

```yaml
data:
  eval_format: "fewshot"
  n_shot: 0          # zero-shot
```

与 few-shot 相同流程，但不拼接示例，直接评估。

### 纯文本 Loss 评估

```yaml
data:
  eval_format: "text"
```

使用原始文本计算 next-token prediction loss，不评估 accuracy。

### 评估指标

每次 eval 输出以下指标：

| 指标 | 说明 |
|------|------|
| `{domain}` | 各 domain 的平均 LM loss |
| `{domain}_ppl` | 各 domain 的 perplexity (exp(loss)) |
| `weighted` | 加权平均 loss |
| `weighted_ppl` | 加权平均 perplexity |
| `fewshot_acc` | Few-shot MCQ accuracy (仅 eval_format=fewshot 时输出) |

## 验证集数据格式

验证集为 JSONL 文件，支持两种格式：

**纯文本格式** (用于 LM loss 计算和 PMP 梯度)：
```json
{"text": "Question: ...\nA. ...\nB. ...\nC. ...\nD. ...\nAnswer: A. ..."}
```

**结构化格式** (也支持)：
```json
{"question": "...", "choices": ["...", "...", "...", "..."], "answer": "A", "subject": "math"}
```

当 `eval_format=fewshot` 时，系统自动从纯文本格式解析出结构化字段。

## 多领域验证集

支持配置多个验证领域，各自独立计算 loss/PPL/accuracy，并按权重加权汇总：

```yaml
data:
  dev_domains:
    - name: "math"
      dir: "data/dev/math"
      weight: 0.5
    - name: "code"
      dir: "data/dev/code"
      weight: 0.3
    - name: "general"
      dir: "data/dev/general"
      weight: 0.2
```

## 主要配置项

| 配置路径 | 默认值 | 说明 |
|----------|--------|------|
| `model.path` | - | HuggingFace 模型路径 |
| `model.max_length` | 2048 | 最大 token 长度 |
| `data.train_dir` | - | 训练数据目录 |
| `data.dev_dir` | - | 验证数据目录 |
| `data.eval_format` | fewshot | 评估格式: fewshot / text |
| `data.n_shot` | 5 | Few-shot 示例数 (0=zero-shot) |
| `clustering.method` | minibatch | 聚类方法: minibatch / kmeans / faiss / random |
| `clustering.cluster_size` | 100 | 每聚类目标样本数 |
| `clustering.kmeans.feature` | intermediate | 特征类型: intermediate / projection / embedding / ghost |
| `clustering.kmeans.embed_layer` | -1 | intermediate 模式的层号 (-1=中间层) |
| `projection.dim` | 1024 | 随机投影维度 |
| `pmp.window_size` | 20 | 环形缓冲区大小 |
| `pmp.update_interval` | 100 | PMP 权重更新间隔 (步) |
| `pmp.temperature` | 1.0 | Softmax 温度 (越大越均匀) |
| `training.total_iters` | 10000 | 总训练步数 |
| `training.lr` | 3e-5 | 学习率 |
| `training.eval_interval` | 500 | 评估间隔 (步) |
| `deepspeed.enabled` | true | 启用 DeepSpeed ZeRO-3 |
| `deepspeed.config_file` | configs/ds_zero3.json | DeepSpeed JSON 配置文件路径 |

完整配置见 `configs/default.yaml`。

## 聚类方法

支持四种聚类后端，通过 `clustering.method` 配置切换：

| 方法 | 说明 | 适用场景 |
|------|------|----------|
| `minibatch` | sklearn MiniBatchKMeans (默认) | 通用，速度快，内存友好 |
| `kmeans` | sklearn 完整 KMeans | 追求聚类精度，数据量较小 |
| `faiss` | Faiss GPU KMeans | 大规模数据 (>100 万样本)，需安装 `faiss-gpu` |
| `random` | 随机分配 | Baseline 对比实验 |

```bash
# 使用 Faiss GPU KMeans
deepspeed --num_gpus=4 train.py --config configs/default.yaml \
    clustering.method=faiss

# 使用完整 KMeans
deepspeed --num_gpus=4 train.py --config configs/default.yaml \
    clustering.method=kmeans
```

## Ghost Projection

可选的参数掩码策略，用于降低梯度投影的计算开销：

- **layerwise**: 按层掩码，只保留指定层的参数
- **random**: 随机掩码一部分参数
- **frequency**: 按更新频率自适应掩码

可分别在聚类阶段 (`clustering.ghost`) 和 PMP 阶段 (`pmp.ghost`) 独立配置。

## Ghost Inner Product 快速路径

设置 `pmp.ghost_ip.enabled=true` 跳过环形缓冲区遍历，使用 Ghost Inner Product 实现 O(1) 复杂度的逐步权重更新：

```yaml
pmp:
  ghost_ip:
    enabled: true
    proj_dim: 1024
    strategy: "random"
    fraction: 0.5
```

## DeepSpeed ZeRO-3

默认启用 DeepSpeed ZeRO-3，将模型参数、梯度和优化器状态分片到多张 GPU 上，大幅降低单卡显存占用。

提供两种预置配置：

| 配置文件 | 说明 | 适用场景 |
|----------|------|----------|
| `configs/ds_zero3.json` | ZeRO-3 纯 GPU (默认) | GPU 显存充足 |
| `configs/ds_zero3_offload.json` | ZeRO-3 + CPU Offload | GPU 显存不足，用 CPU 内存换速度 |

切换配置：

```bash
# 纯 GPU (默认)
deepspeed --num_gpus=8 train.py --config configs/default.yaml

# CPU Offload
deepspeed --num_gpus=8 train.py --config configs/default.yaml \
    deepspeed.config_file=configs/ds_zero3_offload.json
```

**注意事项：**
- DeepSpeed 模式下 checkpoint 保存同时包含 DeepSpeed 原生格式和 HuggingFace 格式 (`hf_model/` 子目录)
- PMP backward 阶段会自动 gather ZeRO-3 分片参数，计算完毕后释放
- 如需关闭 DeepSpeed 回退到 DDP，设置 `deepspeed.enabled=false` 并用 `torchrun` 启动
